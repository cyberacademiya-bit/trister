# Тристер 🤖📈

Торговый бот. Стратегия: **сначала демо (paper trading), потом реал.**
Один и тот же код торгует на демо и на реале — переключается флагом `TESTNET` в [config.py](config.py).

## Философия
- **Не слить важнее, чем заработать.** Риск-менеджмент — приоритет №1.
- **Честный бэктест — закон.** Всегда сравниваем с «купил и держи» (buy & hold). Стратегия хуже B&H = мусор.
- **Без заглядывания в будущее.** Входим на следующей свече после сигнала, не на текущей.
- LLM/AI — это **советник поверх** быстрого ядра, а не тот, кто жмёт buy/sell.

## Фазы проекта
| Фаза | Что | Статус |
|---|---|---|
| 0. Данные | Тянем свечи с биржи | ✅ готово |
| 1. Бэктест | Прогон стратегии по истории + метрики | ✅ готово |
| 2. Демо-бот | Тот же код торгует на Binance/Bybit **Testnet** | ⬜ дальше |
| 3. LLM-слой | Claude-советник: фильтр сделок, режим рынка, объяснения | ⬜ |
| 4. Анализ | Честная статистика демо (winrate, просадка, Sharpe) | ⬜ |
| 5. Реал | Только после стабильных 2-3 мес. на демо | ⬜ |

## Как запустить
```bash
source .venv/bin/activate
python run_backtest.py
```
Крутить настройки (пара, таймфрейм, периоды SMA, комиссия) — в [config.py](config.py).

## Файлы
- `config.py` — все настройки в одном месте
- `data.py` — загрузка и кэш исторических свечей (ccxt)
- `strategy.py` — логика сигналов (пока: SMA crossover)
- `backtest.py` — движок симуляции + метрики
- `run_backtest.py` — запуск и отчёт

## Метрики отчёта
- **Доход стратегии** vs **Buy & Hold** — главное сравнение
- **Winrate** — % прибыльных сделок
- **Макс. просадка** — самое глубокое падение капитала от пика (сколько боли терпеть)
- **Sharpe** — доходность на единицу риска (>1 хорошо, >2 отлично, но на крипте редкость)
